Introduction
OptiLoad is a custom web-based application, developed as a project to support reasearch of my master's thesis. The OptiLoad application implements multi-criteria analysis methods, allowing for the evaluation of package attractiveness based on multiple independent criteria and their assigned weight coefficients. Based on the resulting objective function value, packages can be ranked in terms of their usefulness from the entrepreneur's perspective. In the next stage, the application groups packages according to delivery distance, enabling the loading sequence to be organized in a way that maximizes logistical efficiency â€” packages with more distant delivery addresses are placed deeper within the cargo space. The subsequent stage involves simulating the placement of packages within the vehicle's cargo space, taking into account both geometric constraints and the maximum allowable load weight. The software allows users to generate packages, edit cargo space parameters, select evaluation criteria, and define the corresponding weights and utility function coefficients. The final results of the simulation are presented in the form of a 3D visualization, facilitating the assessment of both the correctness and effectiveness of the proposed loading arrangement. The use of the OptiLoad application enables not only the verification of the theoretical assumptions described in the thesis but also the practical illustration of how the proposed optimization approach operates under conditions close to those found in real-world scenarios. It is not a proper optimization software meant for commercial use by any means.
Example used in master's thesis
In case you want to recreate the example that was used in thesis without having to input all data yourself, it is possible by simply using the button below. Beware that this will skip all the steps required to perform simulation and you will be immidiately redirected to results.
1. Generate stock
To begin optimization process it is necessary to generate warehouse stock that can later be used to run the simulation. It is required as virtually created packages are the core of allowing this simulation's optimization process to be continued. There are two options available: - automatic generation - the stock is being generated with randomized properties and values. A hundred of virtual packages are created when choosing this option. - manual generation - the stock is being generated by user with properties and values set by them. User manually chooses dimensions, weight, delivery time left, distance and amount of packages to be generated. Both options can be combined.
2. Define criteria
Since the optimization process is based on a multi-criteria analysis method, it is necessary to define the evaluation criteria in advance. This enables the process to be more tailored and aligned with the user's specific needs by assigning relative importance to each criterion. However, certain rules must be observed when assigning values: - Each criterion must have a value between 0 and 1 - A value within this range reflects the relative importance of the criterion, with higher values indicating greater significance. - The total sum of assigned values must not exceed 1 - While individual criteria can be assigned arbitrary values within the allowable range, the cumulative total must remain less than or equal to 1. - The sum of defined criteria may be less than 1 - In such cases, the remaining unallocated value will be automatically assigned to a placeholder category labeled "Other factors" This category accounts for potentially relevant factors not explicitly specified by the user. Nevertheless, it is strongly recommended that users define as many relevant criteria as possible to ensure more precise and personalized optimization results.
  • Volume - packages with larger volumes are assigned higher priority.
  • Weight - heavier packages are given greater priority
  • Remaining delivery time - packages requiring faster delivery are prioritized.
  • Distance - packages with more distant destinations receive higher priority.

3. Choose utility function parameters
Due to criteria having different units of measurement it is neccessary to transform them into scalar values allowing calculations to be done. This step is optional as program will automatically assign values to proceed forward, however it is recommended for user to set their own values within reason for a much more precise and personalized result. Attention: If there are no options below, firstly define criteria in step 2.
4. Perform selection optimization process
Once all the steps have been completed, the optimization process for selecting the packages to be loaded onto the vehicle can begin. This process compiles all available information regarding the generated stock, the defined criteria, and the parameters of the utility functions. It then proceeds to calculate the priority of each package, with the objective of maximizing user benefit in accordance with their specified preferences.
